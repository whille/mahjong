# PRD: 网页单机版麻将游戏

> 产品需求文档 - 基于 Ralph 格式设计的迭代开发流程

---

## 1. 项目概述

### 1.1 简介

创建一个网页单机版麻将游戏，玩家可以与AI对手进行经典的四人麻将对战。游戏采用简约现代的视觉风格，支持完整的吃、碰、杠、胡牌规则，并提供多种AI难度选择。

### 1.2 核心目标

| 目标 | 优先级 | 说明 |
|------|--------|------|
| 完整四人麻将规则 | P0 | 吃、碰、杠、胡牌 |
| 3种AI难度 | P0 | 简单/中等/困难 |
| 胡牌自动检测和算分 | P0 | 核心体验 |
| 流畅操作体验 | P1 | 动画、反馈 |
| 响应式设计 | P1 | 桌面+移动端 |

### 1.3 非目标

- ❌ 网络对战功能
- ❌ 多人本地同屏模式
- ❌ 保存/加载游戏进度
- ❌ 社交功能（好友、排行榜等）
- ❌ 抽卡/氪金系统

| 类型 | 选型 | 说明 |

## 4. 数据模型

### 4.1 Tile（牌）

```javascript
{
  id: string,      // 唯一标识: "wan1_0", "sou5_2"
  type: string,    // 牌类型: "wan1"-"wan9", "sou1"-"sou9", "pin1"-"pin9",
                   //          "east", "south", "west", "north",
                   //          "zhong", "fa", "bai"
  name: string     // 中文名: "一万", "红中" 等
}
```

### 4.2 Player（玩家）

```javascript
{
  id: number,           // 0=玩家, 1-3=AI
  hand: Tile[],         // 手牌（最多14张）
  pool: Tile[],         // 弃牌区
  melds: Meld[],        // 副露（吃碰杠）
  isTenpai: boolean,    // 是否听牌
  isHu: boolean         // 是否已胡
}
```

### 4.3 Meld（副露）

```javascript
{
  type: 'pong' | 'kong' | 'chow',
  tiles: Tile[]
}
```

### 4.4 GameState（状态机）

```
WAITING → DEALING → DRAWING → DISCARDING → RESPONDING → GAMEOVER
                        ↑                        │
                        └────────────────────────┘
```
